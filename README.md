## Кофейня
## Кофейня представляет собой место, куда люди приходят для того, чтобы выпить кофе, купить что-нибудь вкусное и просто хорошо провести время.
## В кофейне существует система бонусов. Чтобы начать накапливать бонусы, клиент должен скачать приложение и зарегистрироваться в нем, указав номер телефона. 
* Пользователи
* Поставщики
* Продукция
* Категории продукции
* Журналирование действий пользователя
* Продажи
* Отзывы

# Требрования
## Пользователи
Пользователи - одна из главных сущностей проекта. Она включает в себя клиентов, администраторов и бариста, которые имеют разный уровень доступа к функционалу программной части проекта. Администраторы могут добавлять продукцию, поставщиков, отслеживать действия пользователей (журналирование), смотреть историю продаж, назначать роли, например, дать пользователю возможности бариста или администратора. Бариста могут смотреть список пользователей, продукции и продаж. Обычный пользователь может посмотреть сколько у него бонусов и какие заказы он делал.
### Таблица сущности "Пользователи"

|  Имя поля         | Описание                   | Дополнительная информация   |
| ----------------- |:--------------------------:| ------------------ |
| id                |INTEGER NOT NULL             | Автоинкримент
| Имя               |VARCHAR(100) NOT NULL       |
| Фамилия           |VARCHAR(100) NOT NULL       |
| Количество_бонусов|INTEGER NOT NULL            |
| Номер_телефона    |VARCHAR(13) NOT NULL UNIQUE |Для входа в аккаунт
| Роль              |VARCHAR(20) NOT NULL        |Клиент, Администратор или Бариста

## Поставщики
Поставщики - сущность проекта, которая представляет собой информацию о поставщиках, с которыми сотрудничает кофейня и какую категорию продукции поставщик предоставляет, а также срок, на который заключён контракт.Несколько поставщиков могут поставлять несколько категорий продукций, поэтому здесь связь многие-ко-многим.
### Таблица сущности "Поставщики"

|  Имя поля         | Описание                   | Дополнительная информация   |
| ----------------- |:--------------------------:| ------------------ |
| id                |INTEGER NOT NULL             | Автоинкримент
| Название_компании |VARCHAR(100) NOT NULL       |
| Номер_телефона    |VARCHAR(13) NOT NULL |
| Категория_продукции |VARCHAR(20) NOT NULL | Одна из категорий сущности "Категории продукции" FOREIGN KEY
| Срок_контракта    |INTEGER NOT NULL |
| Дата_заключения_контракта    |DATE |

## Продукция
Продукция - сущность проекта, которая представляет собой информацию о продукции, которую можно приобрести в кофейне, к какой категории она относится, какова её цена , есть ли она в наличии и её принадлежность( собвестенного производства или от поставщиков). Несколько единиц продукции могут относится к одной категории продукции, поэтому здесь связь многие-к-одному.
### Таблица сущности "Продукция"

|  Имя поля         | Описание                   | Дополнительная информация   |
| ----------------- |:--------------------------:| ------------------ |
| id                |INTEGER NOT NULL             | Автоинкримент
| Название_продукции |VARCHAR(100) NOT NULL       |
| Категория_продукции |VARCHAR(20) NOT NULL | FOREIGN KEY
| Цена   |INTEGER NOT NULL | Цена продукции
| Статус | VARCHAR(20) NOT NULL | В наличии или отсутствует

## Категории продукции
Категории продукции - сущность проекта, которая представляет собой информацию о категориях продукции.
### Таблица сущности "Категории продукции"

|   Имя поля        |   Описание              | Дополнительная информация    |
| ---------------- | :--------------------------: | ------------------ |
| id               |INTEGER NOT NULL             |Автоинкримент|
| Название_категории |VARCHAR(100)            |        |

## Отзывы
Для того, чтобы другие люди узнавали о качестве кофейни, необходима система отзывов, которая будет включать в себя оценку от зарегистрированного пользователя, его комментарий и имя. Также для отображения отзыва необходима проверка администратором.
### Таблица сущности "Отзывы"

|  Имя поля         | Описание                   | Дополнительная информация   |
| ----------------- |:--------------------------:| ------------------ |
| id                |INTEGER NOT NULL             | Автоинкримент
| Id_пользователя |INTEGER NOT NULL        | FOREIGN KEY
| Оценка   |INTEGER NOT NULL | Оценка от 0 до 5
| Текст |VARCHAR(255)             | Коментарий, который оставляет пользователь  |

## Продажи
Эта сущность нужна для отображения продажи единицы продукции, несколько продукций могут входить в один заказ, поэтому здесь связь многие-к-одному. 
### Таблица сущности "Продажи"

|   Имя поля        |   Описание              | Дополнительная информация    |
| ---------------- | :--------------------------: | ------------------ |
| id               |BIGINT NOT NULL             |Автоинкримент|
| Id_продукции     |INTEGER NOT NULL            |FOREIGN KEY
| Id_пользователя |INTEGER NOT NULL        | FOREIGN KEY
| Цена            |INTEGER NOT NULL        | FOREIGN KEY
| Количество      |INTEGER NOT NULL        | Количество продаваемой продукции за раз
| Время_заказа |DATETIME(6)       | Когда и во сколько сделан заказ


## Журналирование
Журналирование действий пользователя обеспечивает возможность сохранения изменений в базе данных и восстановления базы данных после любого рода аппаратных и программных сбоев. 
### Таблица сущности "Журналирование"

|   Имя поля        |   Описание              | Дополнительная информация    |
| ---------------- | :--------------------------: | ------------------ |
| id               |BIGINT NOT NULL             |Автоинкримент|
| Сообщение |VARCHAR(500)        | Какое изменение было совершено
| Время |DATETIME(6)       | В какое время было совершено действие
| Сущность |BIGINT NOT NULL        | Сущность, которая подверглась изменению FOREIGN KEY
